2015 · Project

Making Art with NASA Data.

A visual exploration of space and communication using large datasets.

Open Project

TL;DR

Spacelink is an infographic visual experience mapping data requests through NASA’s web systems. It’s built primarily in three.js.

Project Info

  • Team Size: 1 (Design + Eng)
  • Roles: Design, Javascript, Content
  • Deliverable: A single visual piece, created using a large dataset.

Framing the Project

This project is a visual exploration of space and communication. The ideas of space, space travel and communication in space have always fascinated me. I’ve been passionate about science fiction and space exploration since my early school years. As this was primarily an art project I wanted to take a personal approach, exploring a topic that is important to me. The goal of the work is to use a dataset to create art. I struggled to find a dataset that would allow me to create an experience exactly like what I wanted, so I decided to use the data I found as a creative medium rather than to use it as a direction for a narrative.

The visual style of the piece was inspired by old technology. The dataset is from 1995, closer to the glory days of NASA. The movement of the map is designed to reflect tape-based data storage, where data would be recorded as a tape was fed through the system.

Spacelink uses only 136 lines of javascript to create the entire experience.

We rarely stop to consider the huge amount of data that we are constantly sending and receiving.

Communication isn't a constant stream, but something sporadic. Seeing it visually mapped creates interesting and random forms as it moves. The map created is reminiscent of stars. Over time the map scrolls, showing the ever-continuing stream of data that is moving through the system. The piece is fascinating to watch–because it is in constant motion as interesting forms that are observed are temporary.

Of course, since the data is from a set and not live we will eventually see repeating forms and patterns. If I had more time to explore this project I would have loved to bring in live data streams to be mapped. I also would have liked to build out controls that would allow the observer to manipulate the visuals better.

This was my first time working with three.js–I only got a taste of how powerful it is but I’d love to dig into it deeper.

How It’s Built

The dataset I used to generate this piece is a TSV from NASA detailing communications made on their network in 1995. Originally I wanted to find a set of data with communications between mission control and spacecrafts, but I couldn't find anything like that unfortunately. I decided to instead use the data I had to generate a map. I had some technical issues at first. I was attempting to map nodes to DOM elements, but quickly (less than 2000 nodes) the browser became sluggish and unresponsive. I then switched to SVG mapping, which allowed me to map over 35 thousand elements simultaneously.
